# 🚀 الدليل الشامل لمبرمجي التطبيقات الصغيرة والسريعة (Rust & GUI)

هذا الملف يحتوي على خارطة الطريق لفهم كيف تدار واجهات البرامج الحديثة بعيداً عن البرامج الضخمة مثل Electron.

---

## 1. المحركات الأساسية (The Graphics Motors / APIs)
هذه هي اللغات التي يتحدث بها البرنامج مع كرت الشاشة (GPU) لرسم الواجهة.

### ⬅️ OpenGL (القديم المستقر)
* **الوظيفة:** معيار عالمي قديم لرسم الرسومات 2D و 3D.
* **المميزات:** يعمل على كل الأجهزة القديمة والجديدة، مستقر جداً، لا يحتاج تعريفات خاصة.
* **العيوب:** أبطأ قليلاً في المعالجات الحديثة لأنه لا يستغل كامل قوة كرت الشاشة.
* **متى تستخدمه؟** عندما تريد لبرنامجك أن يعمل على أي كمبيوتر مهما كان قديماً.

### ⬅️ Vulkan (المستقبل السريع)
* **الوظيفة:** الجيل الجديد البديل لـ OpenGL.
* **المميزات:** أداء فائق السرعة، استهلاك أقل للبطارية، وصول مباشر للعتاد (Hardware).
* **العيوب:** معقد في البرمجة، قد لا يعمل على كروت الشاشة القديمة جداً.
* **متى تستخدمه؟** عندما تبني تطبيقاً رسومياً ثقيلاً وتريد أقصى استجابة (High Performance).



---

## 2. المكتبات البرمجية (The GUI Libraries)
بما أن المحركات (GL/Vulkan) معقدة، نستخدم "مكتبات" وسيطة تسهل علينا رسم الأزرار والنصوص.

### ⬅️ Slint (الأكثر توازناً واحترافية)
* **اللغة:** Rust / C++ / JavaScript.
* **المحرك:** يدعم OpenGL، Vulkan، وحتى البرمجيات البرمجية (Software Rendering).
* **التصميم:** يستخدم لغة تصميم خاصة تشبه CSS تسمى `.slint`.
* **الحجم:** صغير جداً (حوالي 8-12 ميجابايت).
* **المميزات:** يمنحك تطبيقاً "أصلياً" (Native) تماماً، سريع جداً، ويدعم الأجهزة الضعيفة.



### ⬅️ Tauri (بديل Electron الصغير)
* **الفكرة:** يستخدم Rust للعمليات الخلفية، و HTML/CSS للواجهة.
* **الحجم:** صغير جداً (5-10 ميجابايت) لأنه يستخدم محرك الويب الموجود في ويندوز (Edge/WebView2).
* **المميزات:** إذا كنت تحب تصميم الويب وتريد أصغر حجم ممكن.

### ⬅️ Egui (الأصغر والأبسط)
* **الفكرة:** مكتبة "Immediate Mode"، ترسم الواجهة في كل إطار.
* **الحجم:** قد يصل لـ 2 ميجابايت فقط!
* **المميزات:** رائعة لأدوات المطورين والبرامج التقنية البسيطة.

---

## 3. مقارنة الأحجام والتقنيات (The Showdown)

| التقنية | حجم التطبيق (EXE) | استهلاك الرام (RAM) | أسلوب التصميم |
| :--- | :--- | :--- | :--- |
| **Electron** | 200MB - 300MB | عالي جداً | Web (HTML/CSS) |
| **Tauri** | **5MB - 10MB** | منخفض | Web (HTML/CSS) |
| **Slint** | **8MB - 12MB** | منخفض جداً | Slint DSL (يشبه CSS) |
| **C/C++ Native** | 1MB - 5MB | أقل ما يمكن | معقد يدوياً |

---

## 4. كيف تختار "موتور" العرض؟ (The Back-ends)
عندما تقوم بتحميل برامج مثل **Krokiet/Czkawka**، ستجد خيارات للمحركات:

1.  **all_backends:** البرنامج يختار الأفضل تلقائياً (Vulkan إذا كان متاحاً، وإلا OpenGL).
2.  **skia_opengl:** يستخدم مكتبة Skia للرسم عبر محرك OpenGL (الأكثر استقراراً).
3.  **skia_vulkan:** يستخدم مكتبة Skia للرسم عبر محرك Vulkan (الأسرع للأجهزة الحديثة).

---

## 5. نصائح للمبرمج (للحصول على أصغر حجم EXE)
إذا بدأت البرمجة بلغة **Rust**، اتبع هذه الخطوات لضغط حجم ملفك:

1.  استخدم خيار بناء النسخة النهائية: `cargo build --release`.
2.  أضف هذه الإعدادات في ملف `Cargo.toml` لتقليل الحجم:
    ```toml
    [profile.release]
    opt-level = 'z'     # تحسين الكود لتقليل الحجم
    lto = true          # تحسين وقت الربط
    codegen-units = 1
    panic = 'abort'     # إزالة كود تتبع الأخطاء الزائد
    ```
3.  استخدم أداة `strip` (موجودة في Linux/Git Bash) لمسح معلومات التصحيح: `strip target/release/yourapp.exe`.
